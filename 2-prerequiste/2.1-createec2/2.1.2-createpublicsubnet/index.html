Create Public Subnet

Create Public Subnet

  1. Click Subnets.
  • Click Create subnet.

  1. On the Create subnet page:
  • In the VPC ID field, select Lab VPC.
  • In the Subnet name field, enter Public Subnet 1.
  • In the Availability Zone field, select the first Availability Zone.
  • In the IPv4 CIDR block field, enter 10.10.1.0/24.

  1. Scroll to the bottom of the page and click Create subnet.

  2. Select Public Subnet 1.

  • Click Actions.
  • Click Edit subnet settings.

  1. Select Enable auto-assign public IPv4 address.
  • Click Save.

  1. Click Internet Gateways.
  • Click Create internet gateway.

  1. On the Create internet gateway page:
  • In the Name tag field, enter Lab IGW.
  • Click Create internet gateway.

  1. After successful creation, click Actions.
  • Click Attach to VPC.

  1. On the Attach to VPC page:
  • In the Available VPCs field, select ASG.
  • Click Attach internet gateway.
  • Verify the successful attachment as shown below.

  1. Next, we will create a custom route table to associate with the Public Subnet.
  • Click Route Tables.
  • Click Create route table.

  1. On the Create route table page:
  • In the Name field, enter Route Table Public.
  • In the VPC field, select ASG.
  • Click Create route table.

  1. After successfully creating the route table:
  • Click Edit routes.

  1. On the Edit routes page:
  • Click Add route.
  • In the Destination field, enter 0.0.0.0/0.
  • In the Target field, select Internet Gateway and then select the previously created Lab IGW.
  • Click Save changes.

  1. Click the Subnet associations tab.
  • Click Edit subnet associations to associate the custom route table we just created with the Lab Public Subnet.

  1. On the Edit subnet associations page:
  • Select Lab Public Subnet.
  • Click Save associations.

  1. Verify that the route table has been associated with the Lab Public Subnet and that the route to the internet is directed to the Internet Gateway as shown below.
